# [PRD] 팀 운동 & 식단 챌린지 랭킹 앱 (Team Fit-Track)

## 1. 프로젝트 개요
* **프로젝트명:** 오운완(오늘 운동 완료) 팀 챌린저
* **목적 및 비전:** 팀원들이 매일 운동과 식단을 사진으로 인증하고, 목표 달성률과 인증 점수를 합산해 선의의 경쟁을 펼치는 커뮤니티 앱.
* **타겟 사용자:** 함께 운동하며 동기부여를 얻고 싶은 소규모 크루 또는 팀.
* **MVP 범위:** 목표 설정, 사진 업로드(자동 변환), 점수 자동 계산, 실시간 팀 랭킹 대시보드.

## 2. 문제 정의 및 솔루션
* **문제:** 혼자 하는 운동은 포기하기 쉽고, 메신저로만 인증하면 기록이 쌓이지 않아 랭킹 확인이 어려움. 고화질 사진 업로드 시 저장 용량과 속도 저하 문제 발생.
* **솔루션:** 사진 업로드 즉시 **WebP** 변환으로 최적화하며, 인증 여부를 판단해 점수를 부여하고 실시간 순위를 노출.
* **성공 지표:** 팀원 전체의 일일 인증률 80% 이상 유지, 한 달 챌린지 완주.

## 3. 사용자 스토리 및 요구사항
| ID | 사용자 스토리 | 수용 기준 | 구분 |
| :--- | :--- | :--- | :--- |
| US-001 | 사용자는 자신의 운동 목표(체중 등)를 설정할 수 있다. | 숫자 기반 목표 설정 및 현재 상태 입력 기능. | MVP |
| US-002 | 사용자는 식단/운동 사진을 업로드할 수 있다. | 업로드 시 자동으로 WebP 변환 및 저장 처리. | MVP |
| US-003 | 사용자는 팀원들의 기록을 확인하고 수정할 수 있다. | 전체 공유 데이터베이스 기반의 리스트 뷰 및 수정 버튼. | MVP |
| US-004 | 사용자는 실시간 순위를 확인할 수 있다. | (목표 달성 점수 + 인증 점수) 합산 랭킹 노출. | MVP |

## 4. 기능적 요구사항
* **이미지 최적화 파이프라인:** 사용자가 사진을 올리면 클라이언트 측에서 WebP 파일로 자동 변환하여 저장.
* **자동 채점 엔진:**
    * **목표 점수(Max 100):** (현재 진행치 / 목표치) × 100 실시간 계산.
    * **인증 점수:** 사진 1회 업로드당 10점씩 누적 합산.
* **공유 편집 시스템:** 별도의 복잡한 권한 제한 없이 팀원 모두가 데이터를 수정할 수 있는 오픈 DB 구조.

## 5. 비기능적 요구사항
* **사용성:** 비개발자 사용자를 위해 버튼 크기를 크게 하고, 직관적인 아이콘 사용.
* **성능:** WebP 변환을 통해 이미지 로딩 속도를 1초 이내로 유지.
* **보안:** 팀 전용 비밀번호를 통한 간단한 진입 제한.

## 6. 기술 아키텍처
* **프론트엔드:** React + Vite + TypeScript + TailwindCSS
* **이미지 처리:** 브라우저 Canvas API로 WebP 변환 (클라이언트 사이드)
* **데이터베이스:** MVP는 localStorage 기반 단일 테이블 (팀원별 점수 및 이미지 dataURL 저장)
* **상태 관리:** Zustand 또는 React Context

## 7. UI/UX 가이드라인
* **디자인 원칙:** "Simple & Clean"
* **폰트:** Pretendard (가독성 높은 샌스세리프)
* **색상:** 흰색 배경(#FFFFFF), 강조색은 일렉트릭 블루 1개만 사용.
* **화면 구성:**
    1. **메인 탭:** 현재 우리 팀 랭킹 Top 3와 내 현재 점수.
    2. **인증 탭:** 큰 '사진 찍기' 버튼과 오늘 업로드한 WebP 이미지 미리보기.
    3. **기록 탭:** 팀원 전체의 기록 리스트 (수정 아이콘 포함).

## 8. 개발 우선순위
* **Phase 1 (MVP):** 사진 업로드 & WebP 변환, 기본 점수 계산 로직, 랭킹 보드.
* **Phase 2:** 식단 칼로리 자동 분석, 목표 달성 시 축하 애니메이션.
* **Phase 3:** 월별 기록 리포트 PDF 다운로드 기능.

## 9. 리스크 및 제약사항
* **수정 권한:** 누구나 수정 가능하므로 실수로 데이터를 삭제할 리스크 존재 → '삭제 확인' 팝업 필수 적용.
* **데이터 영속성:** localStorage 기반이므로 브라우저 캐시 삭제 시 데이터 유실 → 향후 백엔드 연동 고려.
